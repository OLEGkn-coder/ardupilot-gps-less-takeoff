# Реалізація зльоту (TKOFF) без GPS для ArduPlane

Цей патч модифікує стандартну логіку режиму AUTO TAKEOFF у прошивці ArduPilot (ArduPlane), дозволяючи здійснювати безпечний старт без наявності валідного 3D GPS Fix. В якості альтернативних джерел даних використовуються датчик повітряної швидкості (Airspeed) та акселерометр (IMU).

## Що було змінено (Логіка роботи)

1. **Обхід блокування ARM (Pre-arm checks):**
   У файлі `takeoff.cpp` змінено умови запуску алгоритму зльоту. Якщо GPS відсутній, система перевіряє наявність заданої вручну точки відліку (`EKF Origin`) та наявність активного датчика повітряної швидкості. Якщо ці дві умови виконані система дозволяє старт алгоритму.

2. **Детекція кидка та набір швидкості:**
   - Стартова розкрутка двигунів (`TKOFF_THR_MIN`) та детекція кидка за прискоренням (`TKOFF_THR_MINACC`) залишені зі стандартної логіки TECS.
   - **Важливо:** Замість швидкості відносно землі (`gps.ground_speed()`), яка дорівнює нулю без GPS, інтегровано отримання швидкості через `ahrs.airspeed_estimate(&current_speed)`. Ця швидкість використовується для перевірки успішності кидка та для запобігання хибному спрацюванню `takeoff_timeout` (вимкнення моторів у повітрі).

3. **Завершення зльоту (Стейт-машина):**
   У файлі `mode_takeoff.cpp` (`ModeTakeoff::update()`) додано примусовий перехід у режим кружляння (`CIRCLE`). Після того, як барометрична висота досягає значення параметра `TKOFF_ALT`, викликається метод `plane.set_mode(Mode::Number::CIRCLE, ModeReason::TAKEOFF_COMPLETE)`.
